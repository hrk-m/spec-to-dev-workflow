# list-users — ユーザー一覧取得

## 概要

ユーザーの一覧を検索キーワードとページネーション条件を指定して取得する。画面表示時に自動で実行され、検索のたびに再取得が走る。クライアントは最大 500 件をキャッシュし、表示件数の切り替えや Previous / Next ページ操作はキャッシュから再計算する（不足時は追加フェッチを自動実行する）。

---

## 処理フロー（正常系）

```
画面が表示される
  │
  ├─ useUserList が limit=500, offset=0 で API リクエストを発行する
  ├─ 初回読み込み時はスケルトンローディングを表示する
  ├─ バックエンドからユーザー一覧（id, first_name, last_name）と total が返る
  ├─ cachedUsers にキャッシュされ、currentPage と perPage で表示ユーザーをスライスして表示する
  └─ リストにユーザー（姓・名）が表示される

検索キーワードを入力する
  │
  ├─ 300ms デバウンス後に cachedUsers をクリアして再フェッチ（q パラメータ付き）
  ├─ effectiveTotal = cachedUsers.length（検索中はキャッシュ件数を使用）
  └─ フィルター結果が表示される

Previous / Next ボタンを押す / 表示件数ボタン（20 / 50 / 100）を押す
  │
  ├─ キャッシュから表示範囲を再計算する（サーバーへの再リクエストなし）
  ├─ キャッシュが不足する場合は追加フェッチが自動実行される（offset=500 で次バッチ）
  └─ ページ情報（現在ページ・総ページ数）が更新される
```

---

## 処理フロー（異常系）

```
バックエンドへの通信が失敗する（ネットワークエラー・500 など）
  │
  └─ エラーメッセージが画面に表示される。一覧は空のまま

検索結果が 0 件の場合
  │
  ├─ isEmptyResult = true になる
  ├─ ヘッダーサブタイトルに「No users found」が表示される
  └─ ページネーション（Previous / Next ボタン）が非表示になる
```

---

## 使用コンポーネント・状態

| 要素 | 種別 | 役割 |
|---|---|---|
| `UsersPage` | コンポーネント | ユーザー一覧ページのルートコンポーネント |
| `UserList` | コンポーネント | 検索・ページネーション・スケルトン・空状態表示付きユーザー一覧コンポーネント |
| `useUserList` | カスタム Hook | fetch・検索・表示件数・ページネーション・キャッシュの状態と処理を管理する |
| `fetchUsers` | API 関数 | `GET /api/v1/users?q=&limit=&offset=` を呼び出す |
| `cachedUsers` | state | サーバーから取得したユーザーをキャッシュする（最大 500 件単位でフェッチ） |
| `total` | state | ユーザーの総件数を保持する（q フィルターなしの全件数） |
| `currentPage` | state | 現在のページ番号を保持する |
| `perPage` | state | 1 ページあたりの表示件数（20 / 50 / 100）を保持する |
| `searchQuery` | state | 検索キーワードを保持する |
| `debouncedQuery` | state | 300ms デバウンス済みの検索キーワード（API リクエストに使用） |
| `isEmptyResult` | derived | ローディング終了後に表示件数が 0 の場合 true になる |
| `effectiveTotal` | derived | 検索中は cachedUsers.length、非検索時は API の total を使用する |
| `totalPages` | derived | effectiveTotal と perPage から算出する |

## 確認観点

```
- [ ] 画面を開くとスケルトン表示後にユーザー一覧が表示される
- [ ] ユーザーの姓・名が正しく表示される
- [ ] 初回マウント時に GET /api/v1/users?limit=500&offset=0 が呼び出される
- [ ] ヘッダーサブタイトルにユーザー総件数（total）が表示される
- [ ] 検索ボックスにキーワードを入力すると 300ms 後に一覧が絞り込まれる
- [ ] 検索をクリアすると全件が再表示される
- [ ] 表示件数ボタン（20 / 50 / 100）で表示件数が切り替わる
- [ ] Next ボタンで次ページのユーザーが表示される
- [ ] Previous ボタンで前ページのユーザーが表示される
- [ ] 最初のページでは Previous ボタンが無効になる
- [ ] 最後のページでは Next ボタンが無効になる
- [ ] バックエンドへの通信が失敗するとエラーメッセージが表示される
- [ ] 検索結果が 0 件のとき「No users found」がヘッダーサブタイトルに表示される
- [ ] 検索結果が 0 件のときページネーション（Previous / Next ボタン）が非表示になる
```
