# サービス概要

spec-to-dev-workflow は、仕様（spec）から開発（dev）へのワークフローを管理するサービスです。

バックエンド API（`sample-api`）とフロントエンド（`sample-front`）の2つのシステムで構成されており、仕様書を起点に実装・テストまでを一貫して管理します。

## 共通レイアウト

全画面に共通して表示される UI コンポーネント。

| コンポーネント | 役割                                                                                                                                                                                                                                  |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Header         | グローバルナビゲーション。ハンバーガーメニューでサイドバーを開く。右端のアカウントアイコン（`aria-label="Account"`）をクリックするとドロップダウンが開き、ログインユーザーの UUID（フル形式）と氏名（firstName + lastName）を表示する |
| Sidebar        | スライドインのナビゲーションメニュー。"Groups" ボタンでグループ一覧（`/`）に、"Users" ボタンでユーザー一覧（`/users`）に遷移する                                                                                                      |

## 画面一覧

| 画面                                                | 概要                                                                                                           |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| [グループ一覧](./group-list/README.md)              | グループを検索・無限スクロール付きで閲覧し、行クリックで詳細へ遷移する。右上ボタンから新規グループを作成できる |
| [グループ詳細](./group-detail/README.md)            | グループの詳細情報とメンバー一覧を検索・無限スクロール付きで閲覧する                                           |
| [ユーザー一覧](./user-list/README.md)               | ユーザーを検索・無限スクロール付きで閲覧する。行クリックでユーザー詳細へ遷移する                               |
| [ユーザー詳細](./user-detail/README.md)             | 選択したユーザーの id / UUID / 姓名を表示する                                                                  |
| [メンテナンス画面](./service-unavailable/README.md) | API 障害・認証失敗時に表示する画面。API 復旧後は自動でトップへリダイレクト                                     |
