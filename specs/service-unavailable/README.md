# service-unavailable — 画面仕様

## 目的・役割

API が応答できない状態（認証失敗・ネットワークエラー・サービス障害）のとき、ユーザーにメンテナンス中であることを伝える画面。マウント時に `GET /api/v1/me` を再確認し、API が復旧していれば自動的にトップページへ遷移する。

---

## 操作手順

**共通（全ロール）**

1. ProtectedRoute で `GET /api/v1/me` が失敗すると、この画面に自動リダイレクトされる
2. この画面は認証不要のため、直接 URL アクセス可能
3. マウント時に `GET /api/v1/me` を自動実行する
   - 200 が返れば → `/` へ自動リダイレクト（API 復旧済み）
   - エラーが続けば → 「ただいまメンテナンス中です。」を表示する

## 画面遷移

| 遷移元 | 遷移先 | トリガー |
|---|---|---|
| 任意の保護ルート | この画面（service-unavailable） | `GET /api/v1/me` が 401 またはネットワークエラー |
| この画面（service-unavailable） | グループ一覧（`/`） | マウント時の `GET /api/v1/me` が 200 を返す |

---

## 前提条件・制約

- `ProtectedRoute` の外側に配置されているため、認証チェックなしでアクセスできる
- マウント時の API チェックは一回限り（ポーリングなし）

---

## エラー・注意事項

| エラー / 注意 | 内容 |
|---|---|
| `GET /api/v1/me` が継続してエラー | メンテナンス画面を表示し続ける。自動リトライなし |
| ローディング中 | `null` を返し何も表示しない（チラつき防止） |
