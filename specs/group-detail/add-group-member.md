# add-group-member — メンバー追加

## 概要

グループ詳細画面の Members セクションヘッダーにある「メンバー追加」ボタンをクリックすると、AddMemberSheet（シート）が表示される。シート内でグループ未所属のユーザーを検索・選択し、「一括追加」ボタンで一度に複数名をグループメンバーとして追加できる。フルページモード・シートモードの両方で利用できる。

---

## 処理フロー（正常系）

```
ユーザーが「メンバー追加」ボタンをクリックする
  │
  ├─ AddMemberSheet が SheetStack に積まれ、右からスライドインして表示される
  ├─ AddMemberSheet のマウント時に clearNonMemberListCache(groupId) を呼び出して非メンバーキャッシュをクリアする
  ├─ GET /api/v1/groups/:id/non-members?limit=100&offset=0 を送信する（常に最新データを取得）
  ├─ 未所属ユーザー一覧がシート内に表示される
  ├─ ユーザーが検索キーワードを入力する（300ms デバウンス）
  │    ├─ GET /api/v1/groups/:id/non-members?q={keyword}&limit=100&offset=0 を送信する
  │    └─ 検索結果で一覧を更新する
  ├─ ユーザーがヘッダーチェックボックス（全選択）をクリックする
  │    ├─ 全未選択または一部選択（indeterminate）の場合 → 全非メンバーを選択する
  │    └─ 全選択済みの場合 → 全選択を解除する
  │    └─ 非メンバーが 0 件の場合 → ヘッダーチェックボックスは disabled（操作不可）
  ├─ ユーザーがチェックボックス（またはユーザー行）をクリックして個別に追加対象を選択・解除する
  │    └─ 一部のみ選択された場合 → ヘッダーチェックボックスが indeterminate 表示になる
  ├─ 「一括追加」ボタンをクリックする
  ├─ POST /api/v1/groups/:id/members { "user_ids": [...] } を送信する（ボタンが disabled になる）
  ├─ API が 201 Created を返す
  ├─ clearMemberListCache() でメンバー一覧キャッシュをクリアする
  ├─ useGroupDetail.refetch() でグループ詳細（member_count）を更新する
  └─ AddMemberSheet が閉じる（onClose() 経由）
```

---

## 処理フロー（異常系）

```
未所属ユーザー一覧の取得失敗（4xx・5xx）
  │
  └─ シート内にエラーメッセージを表示する。シートは開いたまま

「一括追加」で 409 Conflict
  │
  └─ シート内に「選択したユーザーはすでにメンバーです」を表示する。シートは開いたまま。選択状態は維持される

「一括追加」でその他エラー（4xx・5xx）
  │
  └─ シート内に汎用エラーメッセージを表示する。シートは開いたまま。選択状態は維持される

未選択状態で「一括追加」クリック
  │
  └─ 「一括追加」ボタンは disabled のためクリック不可（送信されない）
```

## 使用コンポーネント・状態

| 要素                 | 種別           | 役割                                                                                                                                                                                                                     |
| -------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `GroupDetailContent` | コンポーネント | 「メンバー追加」ボタンを配置し、`openSheet()` で AddMemberSheet を SheetStack に登録する                                                                                                                                 |
| `AddMemberSheet`     | コンポーネント | 検索入力・チェックボックス付きユーザー一覧・「一括追加」ボタンを提供するシートコンテンツ。レイアウト（上から）：検索入力 → 「一括追加」ボタン（コンテンツ幅・右端揃え・soft スタイル）→ チェックボックス付きユーザー一覧 |
| `useNonMemberList`   | カスタム Hook  | 未所属ユーザー一覧取得・クライアントサイドページネーション・検索クエリ管理を担う。`clearNonMemberListCache(groupId?: number)` でキャッシュをクリアできる（`groupId` 指定時は該当グループのみ、未指定時は全件）           |
| `addGroupMembers`    | API 関数       | `POST /api/v1/groups/:id/members` を呼び出す                                                                                                                                                                             |
| `selectedIds`        | state          | チェックされたユーザー ID の集合（`Set<number>`）                                                                                                                                                                        |
| `submitError`        | state          | 一括追加の API エラーメッセージ                                                                                                                                                                                          |
| `isSubmitting`       | state          | API 呼び出し中かどうか。「一括追加」ボタンの disabled 制御に使用                                                                                                                                                         |
| `headerCheckboxRef`  | ref            | ヘッダーチェックボックスの DOM 参照。`useEffect` で `indeterminate` 属性を直接設定するために使用                                                                                                                         |
| `isAllSelected`      | 派生値         | `users.length > 0 && selectedIds.size === users.length` のとき `true`。ヘッダーチェックボックスの `checked` に反映                                                                                                       |
| `isSomeSelected`     | 派生値         | `selectedIds.size > 0 && selectedIds.size < users.length` のとき `true`。ヘッダーチェックボックスの `indeterminate` に反映                                                                                               |

## 確認観点

```
- [ ] フルページ・シートモードの両方に「メンバー追加」ボタンが表示される
- [ ] ボタンクリックで AddMemberSheet が表示され、未所属ユーザー一覧が描画される
- [ ] 未所属ユーザーの検索（キーワード入力）で一覧が絞り込まれる
- [ ] 検索クリアで未所属ユーザー全件が再表示される
- [ ] ユーザー行をクリックまたはチェックボックスで個別に選択・解除できる
- [ ] ヘッダーチェックボックスをクリックすると全非メンバーが選択される（全未選択または indeterminate 状態から）
- [ ] 全選択状態でヘッダーチェックボックスをクリックすると全解除される
- [ ] 一部選択（indeterminate）状態でヘッダーチェックボックスをクリックすると全選択される
- [ ] 非メンバーが 0 件のときヘッダーチェックボックスが disabled で操作不可
- [ ] ヘッダーチェックボックスで全選択して「一括追加」をクリックするとシートが閉じる
- [ ] 「一括追加」ボタンが検索フォームの直下・ユーザー一覧の上に表示される（右端揃え）
- [ ] 未選択状態では「一括追加」ボタンが disabled
- [ ] 1 件以上選択すると「一括追加」ボタンが enabled になる
- [ ] 選択後に「一括追加」をクリックして成功 → シートが閉じ、メンバー数が増加した値で表示される
- [ ] 複数ユーザーを同時に選択して一括追加できる
- [ ] 409 エラー時はシート内に「選択したユーザーはすでにメンバーです」が表示される（シートは開いたまま・選択状態を維持）
- [ ] その他エラー時はシート内に汎用エラーメッセージが表示される（シートは開いたまま・選択状態を維持）
- [ ] GET /non-members が失敗するとシート内にエラーメッセージが表示される
- [ ] 未所属ユーザーがいない場合に「追加できるユーザーがいません。」が表示される
- [ ] シートモードで「メンバー追加」ボタンをクリックすると AddMemberSheet が 2 枚目のシートとして積まれる
- [ ] メンバー追加後に AddMemberSheet を再度開くと、追加済みユーザーが非メンバーリストに表示されない（マウント時キャッシュクリアにより最新データを取得）
```

## 使用 API

| エンドポイント                   | メソッド | 用途                                                 |
| -------------------------------- | -------- | ---------------------------------------------------- |
| `/api/v1/groups/:id/non-members` | GET      | グループ未所属のユーザー一覧と total を取得する      |
| `/api/v1/groups/:id/members`     | POST     | 選択したユーザーをグループメンバーとして一括追加する |
