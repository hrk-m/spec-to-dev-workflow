# get-group — グループ詳細取得

## 概要

指定されたグループの詳細情報（名称・説明・メンバー数）を取得する。画面表示時に自動で実行され、グループ情報カードに反映される。

---

## 処理フロー（正常系）

```
グループ詳細画面が表示される（URL の :id パラメータでグループを特定）
  │
  ├─ useGroupDetail が groupId を元に API リクエストを発行する
  ├─ スケルトンローディングを表示する
  ├─ バックエンドからグループ詳細情報（id・name・description・member_count・subgroups）が返る
  └─ グループ情報カードに名称・説明が表示される
     Members セクションヘッダーにメンバー総件数が表示される
     Subgroups セクションにサブグループ一覧が表示される
```

---

## 処理フロー（異常系）

```
バックエンドへの通信が失敗する（ネットワークエラー・404・500 など）
  │
  └─ エラーメッセージが画面に表示される。グループ情報・メンバーセクションは非表示のまま
```

---

## 使用コンポーネント・状態

| 要素                | 種別           | 役割                                                             |
| ------------------- | -------------- | ---------------------------------------------------------------- |
| `GroupDetailPage`   | コンポーネント | グループ詳細画面全体を構成する                                   |
| `GroupInfoSkeleton` | コンポーネント | グループ情報の読み込み中スケルトンを表示する                     |
| `useGroupDetail`    | カスタム Hook  | グループ詳細の fetch・ローディング・エラーの状態を管理する       |
| `group`             | state          | 取得したグループ詳細情報を保持する                               |
| `subgroups`         | state          | 取得したサブグループ一覧を保持する（`group.subgroups` から派生） |
| `error`             | state          | エラーメッセージを保持する                                       |
| `isLoading`         | state          | ローディング状態を保持する                                       |

---

## 確認観点

```
- [ ] 画面を開くとスケルトン表示後にグループ情報が表示される
- [ ] グループの名称・説明が正しく表示される
- [ ] Members セクションにメンバー総件数が表示される
- [ ] バックエンドへの通信が失敗するとエラーメッセージが表示される
- [ ] 存在しないグループ ID でアクセスした場合にエラーが表示される
- [ ] コンポーネントのアンマウント時にステートが更新されない（メモリリーク防止）
- [ ] Subgroups セクションにサブグループが表示される
- [ ] サブグループが 0 件のとき Subgroups セクションに「サブグループはまだありません」が表示される
- [ ] サブグループが 1 件以上のときグループ名・説明がカードリスト形式で表示される
```

## 使用 API

| エンドポイント       | メソッド | 用途                       |
| -------------------- | -------- | -------------------------- |
| `/api/v1/groups/:id` | GET      | グループ詳細情報を取得する |
