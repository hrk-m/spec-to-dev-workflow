# group-detail — 画面仕様

## 目的・役割

特定のグループの詳細情報とそのメンバー一覧を確認する画面。グループの名称・説明を表示し、所属メンバーを検索・ページネーション付きで閲覧できる。

---

## 識別情報

| 項目 | 内容 |
|---|---|
| 画面タイトル | {グループ名}（動的） |
| 画面 ID | group-detail |
| URL / パス | `/groups/:id` |

---

## 想定利用者

| 項目 | 内容 |
|---|---|
| 対象ユーザー | 全ユーザー |
| 必要な権限・ロール | 認証不要 |
| アクセス制御 | 制限なし |

## レイアウトと主要パーツ

| パーツ | 役割 |
|---|---|
| 戻るボタン（< Groups） | グループ一覧画面へ戻る |
| グループ情報カード | グループの名称・説明を表示する |
| Members セクションヘッダー | 「Members」タイトルとメンバー総件数を表示する |
| メンバー検索ボックス | キーワードでメンバーを名前で絞り込む |
| 表示件数切替ボタン | 1 ページあたりの表示件数（20 / 50 / 100）を切り替える |
| メンバーリスト | イニシャルアバターとフルネームでメンバーを一覧表示する |
| Previous / Next ボタン | メンバー一覧のページを前後に切り替える |
| ページ情報 | 現在のページ番号と総ページ数を表示する |

---

## 操作手順

> この画面は認証不要のため権限分岐なし。

**共通（全ロール）**

1. グループ一覧からグループ行をクリックする → グループ詳細画面が表示される
2. グループの名称・説明が表示される。メンバー一覧がスケルトン表示ののちリストに表示される
3. メンバー検索ボックスにキーワードを入力する → メンバーが名前で絞り込まれる
4. 表示件数ボタン（20 / 50 / 100）をクリックする → 1 ページの表示件数が切り替わる
5. 「Next」ボタンを押す → 次のページのメンバーが表示される
6. 「Previous」ボタンを押す → 前のページのメンバーが表示される
7. 「< Groups」ボタンを押す → グループ一覧画面へ戻る

---

## ボタン・リンクの機能

| ボタン / リンク | 操作後の動作 |
|---|---|
| < Groups（戻るボタン） | グループ一覧画面（`/`）へ遷移する |
| メンバー検索ボックス（入力） | メンバーを名前で絞り込む。ページが 1 にリセットされる |
| 表示件数ボタン（20 / 50 / 100） | 1 ページあたりの表示件数を変更する。ページが 1 にリセットされる |
| Previous | 前ページのメンバー一覧を表示する |
| Next | 次ページのメンバー一覧を表示する |

---

## 画面遷移

| 遷移元 | 遷移先 | トリガー |
|---|---|---|
| グループ一覧（group-list） | この画面（group-detail） | グループ行をクリック |
| この画面（group-detail） | グループ一覧（group-list） | 「< Groups」戻るボタンをクリック |

---

## 入力項目

| 項目名 | 必須 / 任意 | 形式・制約 |
|---|---|---|
| メンバー検索キーワード | 任意 | 自由テキスト。名前（姓・名）が対象 |

---

## 前提条件・制約

- 認証不要。誰でもアクセスできる
- URL パラメータ `:id` でグループを特定する
- メンバー一覧はクライアントサイドでキャッシュし、表示件数の切り替え時にサーバーへ再リクエストしない（大量データ時は追加フェッチで補完する）

---

## エラー・注意事項

| エラー / 注意 | 内容 |
|---|---|
| グループ情報の取得失敗 | エラーメッセージが画面に表示される。グループ情報・メンバー一覧は空のまま |
| メンバー一覧の取得失敗 | エラーメッセージがメンバーセクションに表示される |
| 存在しないグループ ID | エラーメッセージが表示される |
