# 用語集

プロジェクト全体で使用するドメイン用語の正規定義。

`specs/` 配下の仕様書を書く際は、ここに定義された用語を優先して使用する。
新しいドメイン用語が登場したら、この用語集に追記する。

---

## ドメイン用語

| 用語 | 定義 | 補足・使用例 |
|---|---|---|
| グループ | ユーザーをまとめる単位 | API では `group` |
| メンバー | グループに所属するユーザー | API では `member`。姓（`last_name`）と名（`first_name`）を持つ |
| メンバー数 | グループに所属するユーザーの人数 | `member_count` |
| 検索 | キーワードによる一覧の絞り込み操作 | 「絞り込み」「フィルター」は使わない |
| ページネーション | ページ単位で一覧を分割するナビゲーション。API は `offset` / `limit` パラメータで制御する | Previous / Next ボタン、ページ情報。グループ一覧・メンバー一覧で採用 |
| 表示件数 | 1 ページあたりに表示するアイテム数 | `perPage`。20 / 50 / 100 の選択肢 |
| クライアントキャッシュ | サーバーから一括取得したデータをクライアント側で保持し、ページ切り替えや表示件数変更をキャッシュから再計算する方式 | グループ一覧では 500 件単位でフェッチ。メンバー一覧でも同様に採用 |
| イニシャルアバター | メンバーの姓・名の頭文字を表示する丸形アイコン | メンバーリストで使用 |
