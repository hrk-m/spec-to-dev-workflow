# メッセージ取得

## 概要

挨拶メッセージを1件取得する。

---

## 画面

<img src="../img/hello.png" width="400" alt="Hello 画面" />

---

## API エンドポイント

```text
GET /hello
```

### リクエスト

パラメータなし。

### レスポンス

#### 成功（200 OK）

```json
{
  "message": "hello"
}
```

#### エラー（500 Internal Server Error）

```json
{
  "message": "エラーの内容"
}
```

## 処理フロー

```text
フロントエンド                     バックエンド
    |                                  |
    |  GET /hello                      |
    |--------------------------------->|
    |                                  | メッセージを生成する
    |  200 OK { "message": "hello" }   |
    |<---------------------------------|
    |                                  |
    | メッセージを画面に表示する         |
```

## 画面表示ルール

| 状態     | 表示内容                       |
| -------- | ------------------------------ |
| 取得中   | 「loading...」を表示する       |
| 取得成功 | 取得したメッセージを表示する   |
| 取得失敗 | エラーの内容を赤字で表示する   |

## バックエンド実装ポイント

- サービス層でメッセージを生成する
- エラーが発生した場合はステータスコードに対応したエラーレスポンスを返す
- Clean Architecture パターンに従い、`domain` / `service` / `handler` の3層で実装する

## フロントエンド実装ポイント

- 画面が表示されたタイミングで自動的にメッセージを取得する
- 取得中は「loading...」を表示する
- Feature-Sliced Design (FSD) に従い、`pages/home` 配下に実装する
- API 通信は `shared/api/client` の共通クライアントを使用する
