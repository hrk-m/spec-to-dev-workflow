# list-groups — グループ一覧取得

## 概要

グループの一覧を検索キーワードとページ条件を指定して取得する。画面表示時に自動で実行され、検索・ページ操作のたびに再取得が走る。

---

## 処理フロー（正常系）

```
画面が表示される / 検索キーワードを入力する / Load More ボタンを押す
  │
  ├─ useGroupList が search・page・limit を元に API リクエストを発行する
  ├─ 初回読み込み時はスケルトンローディングを表示する
  ├─ バックエンドからグループ一覧とページネーション情報が返る
  ├─ リストにグループ（名称・説明・メンバー数）が表示される
  ├─ グループ行をクリックするとグループ詳細画面（/groups/:id）へ遷移する
  └─ Load More 押下時は次ページの結果が一覧の末尾に追加される
     ページ情報（現在ページ・総ページ数）が更新される
```

---

## 処理フロー（異常系）

```
バックエンドへの通信が失敗する（ネットワークエラー・500 など）
  │
  └─ エラーメッセージが画面に表示される。一覧は空のまま

検索ワード入力中に連続リクエストが発生する
  │
  └─ 最新のリクエスト結果のみ反映される（古いレスポンスは無視される）
```

---

## 使用コンポーネント・状態

| 要素 | 種別 | 役割 |
|---|---|---|
| `GroupList` | コンポーネント | グループ一覧リストと Load More ページネーションを表示する |
| `useGroupList` | カスタム Hook | fetch・search・page・Load More の状態と処理を管理する |
| `groups` | state | 取得したグループの配列を保持する（Load More で追加蓄積される） |
| `pagination` | state | total・page・limit のページネーション情報を保持する |
| `search` | state | 検索キーワードを保持する |
| `page` | state | 現在のページ番号を保持する |
| `hasNextPage` | derived | 次ページが存在するかを判定する |
| `isLoadingMore` | state | Load More 中のローディング状態を保持する |

## 確認観点

```
- [ ] 画面を開くとスケルトン表示後にグループ一覧が表示される
- [ ] グループの名称・説明・メンバー数が正しく表示される
- [ ] 検索ボックスにキーワードを入力すると一覧が絞り込まれる
- [ ] 検索時にページが 1 にリセットされる
- [ ] グループ行をクリックするとグループ詳細画面へ遷移する
- [ ] Load More ボタンで次ページのグループが末尾に追加される
- [ ] 最後のページでは Load More ボタンが非表示になる
- [ ] バックエンドへの通信が失敗するとエラーメッセージが表示される
- [ ] 検索キーワードの連続入力で不整合なデータが表示されない
- [ ] 検索結果が 0 件のとき空メッセージが表示される
```
