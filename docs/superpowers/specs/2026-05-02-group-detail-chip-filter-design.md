# グループ詳細画面 — フィルターチップ型リデザイン

**Date:** 2026-05-02
**Status:** Design Approved

---

## 概要

グループ詳細画面の左サイドパネル（サブグループチェックボックスリスト）を廃止し、メンバーリスト上部にフィルターチップ行を新設する。メンバーリストを全幅表示にしてスペース効率を高める。サブグループの追加・削除は新設の「サブグループ管理シート」に一元化する。

---

## 変更の動機

- 現在のスプリットビューはPC向けで、モバイルでは縦積みになり左パネルが大きなスペースを占める
- サブグループ選択（絞り込み）とサブグループ管理（追加・削除）が同一パネルに混在しており役割が不明確
- メンバーリストが右半分にしか表示されず、UUID・姓名・所属元の列が狭い

## チップ行の仕様

### 表示

- サブグループごとに1チップを表示（名前 + メンバー数）
- チップ行は横スクロール対応（サブグループが多くても崩れない）
- 右端に「サブグループ管理 ›」ボタンを固定表示

### 選択動作

- **複数選択可**：複数チップを同時に ON にできる
- **「すべて」チップなし**：全チップ ON = すべてのメンバー表示と同義

### チップの状態

| 状態 | チップの見た目 | メンバーリストタイトル |
|---|---|---|
| 全チップ ON | 青塗り（`background:#e8f0fe`、`border:#1d6fe8`） | 「すべてのメンバー」|
| 一部 ON | ON = 青塗り、OFF = グレー | 「選択中のメンバー」|
| 全チップ OFF | 全グレー | 「直接メンバー」（直接所属のみ表示） |

### インタラクション

- チップをタップ → ON/OFF トグル
- チップ初期状態：全チップ ON（画面ロード時）
- サブグループが0件の場合：チップ行を非表示にし「サブグループ管理 ›」ボタンのみ表示

## サブグループ管理シート（新設）

「サブグループ管理 ›」ボタンでオープンするボトムシート。

### 構成

```
┌──────────────────────────┐
│ ●●●（ハンドル）           │
│ サブグループ管理           [＋ 追加] │
│ グループ名（小テキスト）    │
├──────────────────────────┤
│ チームA                  │
│ 8 members         [削除] │
├──────────────────────────┤
│ チームB                  │
│ 6 members         [削除] │
└──────────────────────────┘
```

### 動作

- **「＋ 追加」**：`useSheetStack` で `AddSubgroupSheet` をスタック呼び出しする（既存コンポーネントをそのまま流用）
- **「削除」**：既存の `DeleteSubgroupDialog` の確認ダイアログを呼び出す
- 追加・削除が完了したら親画面の `refetch()` を呼びチップ行とメンバーリストを更新する

---

## メンバーリストの変更

- **レイアウト**：全幅表示（左パネル廃止により `splitViewLeft` クラスを削除）
- **列構成**：変更なし（UUID・姓名・所属元）
- **その他機能**：検索・チェックボックス選択・削除ボタンは現状維持
