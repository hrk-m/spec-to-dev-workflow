# /plan — 詳細設計コマンド

ユーザーと対話しながら「何を・なぜ・どう作るか」を決め切り、**詳細設計レベルの処理フロー**を確定する。機能の既存仕様は `specs/` ディレクトリを参照する。

## いつ使うか

| タイミング | 説明 |
| --- | --- |
| 新機能の開発を始める前 | 何を作るか・なぜ作るかをユーザーと合意するために使う |
| 既存機能を拡張・変更する前 | 変更範囲と処理フローをユーザーと確認するために使う |
| `prd-writer` を実行する前 | 要件書き出しに必要な目的・ゴール・フローを確定するために使う |

**使ってはいけないタイミング**:
- すでに目的・ゴール・フローが合意済みの場合（重複になる）
- コードのバグ修正・リファクタリングのみの場合（設計不要）

---

## 実行フロー概要

```
スキル起動
  ↓
実装パターンの把握（自動）
  - `sample-api-agent` に技術スタック・アーキテクチャの調査を依頼する
  - `sample-front-agent` に技術スタック・アーキテクチャの調査を依頼する
  ↓
Phase 1: 目的・ゴールの合意（ユーザーと対話）
  ↓ 合意が取れたら(`AskUserQuestion` でユーザーの最終合意をとる)
Phase 2: 詳細処理フローの設計（ユーザーと対話）
  ↓ 合意が取れたら(`AskUserQuestion` でユーザーの最終合意をとる)
完了報告 → prd-writer スキルへ案内
```

---

## 完了条件

- 機能の目的・ゴールがユーザーと合意されている
- 機能名が確定している
- 正常系・異常系の処理フローが箇条書きで整理されている
- フロントエンド・バックエンドそれぞれの処理フローが正常系・異常系で整理されている
- 条件分岐がテーブルで整理されている
- エラーケースが発生レイヤーを含めて全パターン列挙されている
- ユーザーが詳細フロー確定に同意している

---

## 実行手順（この順序で必ず行う）

### Phase 1 — 目的・ゴールの合意

> **用途**: 対象機能と「なぜ作るか・完成形」をユーザーと合意する。合意なしに Phase 2 へ進めない。

@references/phase1-goal-alignment.md

### Phase 2 — 詳細処理フローの設計

> **用途**: 合意した目的・ゴールをもとに、エンドポイント・処理ステップ・エラーケースを確定する。ユーザーの合意なしに完了とみなさない。
> **実装パターン参照の目的**: API・フロントエンドの技術スタック（使用フレームワーク・言語・ライブラリ）を把握するためのみに使用する。
> 命名規則・ディレクトリ構造・変数名・関数名・レイヤー区別などの具体的な実装詳細は Phase 2 の処理フローに含めない。

@references/phase2-flow-design.md

## 制約

- **Phase 2 確認手順（最優先・絶対厳守）**: Phase 2 に入ったら `phase2-flow-design.md` のステップ 5-1〜5-4 を必ず **1 ステップずつ** `AskUserQuestion` で確認する。全体フローを先に提示することは禁止。完了条件に「フローが整理されている」とあっても、この確認手順を経てから整理すること。
- 不明点は推測で埋めない。必ず `AskUserQuestion` で確認する
- **用語統一**: 同じ概念には同じ用語を使う。独自の呼び方が必要な場合は定義を明記する
- 機能の既存仕様は `specs/` を参照する

## フォールバック

| 状況 | 対応 |
| --- | --- |
| `specs/` に機能が存在しない | options に「新機能を追加する」のみ表示して選択させる |
| 引数が空 | `AskUserQuestion` で対象機能の選択から開始 |
| 目的・ゴールが曖昧 | `AskUserQuestion` で深掘りして合意が取れるまで確定しない |
| 処理ステップに不明点がある | `AskUserQuestion` で確認が取れるまで次フェーズに進めない |
| 回答が空・未選択 | 推測で進めず質問を再提示する |

---

## ツール案内

- **sample-api-agent / sample-front-agent**: スキル起動直後に各ディレクトリの実装パターン（技術スタック・アーキテクチャ）を把握するために使う。コードの参照はこれらのエージェントに委譲する
- **Glob / Read**: Phase 1 の開始時に `specs/` の既存仕様を収集するためのみに使う
- **AskUserQuestion**: Phase 1・2 の各合意ポイントで使う
  - 選択肢のどれも該当しない場合、ユーザーは自動的に表示される **"Other"** を選んで自由テキストを入力できる
  - `options` に "Other" を手動追加する必要はない（ツールが自動付与する）
  - ユーザーが "Other" を選択した場合は、入力内容を要件に反映してから次フェーズへ進む
