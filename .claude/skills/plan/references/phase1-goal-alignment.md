# Phase 1 — 目的・ゴールの合意

1. `Glob` で `specs/*/README.md` を取得し、既存の機能一覧を収集する
2. `AskUserQuestion(multiSelect: true)` で対象機能を **複数選択** させる
   - options に既存機能名を列挙し、末尾に **「新機能を追加する」** を必ず含める
   - 例: `["user-management", "hello", ..., "新機能を追加する"]`
   - 選択された既存機能すべての `specs/{機能名}/README.md` を `Read` して仕様を把握する
   - 「新機能を追加する」が含まれる場合: `AskUserQuestion` で機能名・目的を自由記述で確認する
3. ユーザーの入力から **目的**（なぜ作るか）と **ゴール**（完成した状態）を引き出す
   - 不明な場合は推測せず `AskUserQuestion` で確認する
4. 目的・ゴールをチャットに整理して提示し、ユーザーの合意を得る
   - 合意が取れるまで Phase 2 に進めない
