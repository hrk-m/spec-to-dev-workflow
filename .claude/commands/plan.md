# /plan — 要件定義・設計コマンド

<background_information>

**Mission**: ユーザーと対話しながら「何を・なぜ・どう作るか」を決め切る。

**Success Criteria**:
- 機能の目的・ゴールがユーザーと合意されている
- 機能名が確定している
- 要件が `plans/tasks/{タスク名}/prd.md` に整理されている
- API エンドポイント・リクエスト/レスポンス・処理フローが確定している
- ユーザーがフロー確定に同意してから prd.md を書き出している

</background_information>

<instructions>

## Core Task

`$ARGUMENTS` を起点にユーザーと会話しながら目的・ゴール・フローを段階的に確定し、`prd` スキルで要件を整理して `plans/tasks/{タスク名}/prd.md` に書き出す。機能の既存仕様は `specs/` ディレクトリを参照する。

---

## 実行手順（この順序で必ず行う）

### Phase 1 — 目的・ゴールの合意

1. `Glob` で `specs/*/README.md` を取得し、既存の機能一覧を収集する
2. `AskUserQuestion(multiSelect: true)` で対象機能を **複数選択** させる
   - options に既存機能名を列挙し、末尾に **「新機能を追加する」** を必ず含める
   - 例: `["user-management", "hello", ..., "新機能を追加する"]`
   - 選択された既存機能すべての `specs/{機能名}/README.md` を `Read` して仕様を把握する
   - 「新機能を追加する」が含まれる場合: `AskUserQuestion` で機能名・目的を自由記述で確認する
   - 既存機能と新機能を同時に選択した場合は両方を組み合わせてタスクの文脈を構成する
3. ユーザーの入力から **目的**（なぜ作るか）と **ゴール**（完成した状態）を引き出す
   - 不明な場合は推測せず `AskUserQuestion` で確認する
   - 例: 「この機能で誰が何を解決できますか？」「完成したとき何ができていれば OK ですか？」
4. 目的・ゴールをチャットに整理して提示し、ユーザーの合意を得る
   - 合意が取れるまで Phase 2 に進めない

### Phase 2 — フローの設計と確定

5. 目的・ゴールをもとに API フロー（エンドポイント・リクエスト/レスポンス・処理順序）の案を提示する
6. `AskUserQuestion` でフローの合意を取る
   - 「このフローで進めますか？」「変えたい部分はありますか？」
   - 修正があれば反映して再提示し、合意が取れるまで繰り返す
7. フロー確定後にタスク名を英小文字 kebab-case で決定する
   - 例: `ユーザー管理機能` → `user-management`

### Phase 3 — prd.md への書き出し

8. 合意した内容のみ `plans/tasks/{タスク名}/prd.md` に反映する
9. 要件確定後の次アクションとして `/steering` を明示する

---

## ドキュメント構造

```text
specs/               # 機能仕様の参照元（読み取り専用）
  {機能名}/
    README.md
    *.md

plans/               # /plan コマンドの出力先
  tasks/
    {タスク名}/
      prd.md         # タスク要件
```

---

## 制約

- `prd` スキルを必ず使う
- 不明点は推測で埋めない。必ず `AskUserQuestion` で確認する
- 目的・ゴール・フローの合意が取れるまで prd.md を書き出さない
- 既存 prd.md がある場合は上書きより追記・統合を優先する
- 機能の既存仕様は `specs/` を参照する
- 要件の一次情報は `plans/tasks/{タスク名}/prd.md`

---

## フォールバック

| 状況 | 対応 |
| --- | --- |
| `specs/` に機能が存在しない | options に「新機能を追加する」のみ表示して選択させる |
| `$ARGUMENTS` が空 | `AskUserQuestion` で対象機能の選択から開始 |
| 目的・ゴールが曖昧 | `AskUserQuestion` で深掘りして合意が取れるまで確定しない |
| フローに不明点がある | `AskUserQuestion` で確認が取れるまで次フェーズに進めない |
| 回答が空・未選択 | 推測で進めず質問を再提示する |
| prd.md が未存在 | 新規作成する |

</instructions>

## ツール案内

- **Glob / Read**: 既存 `specs/` の参照（機能仕様の確認）
- **AskUserQuestion**: 目的・ゴール・フローの合意取得（各フェーズで使用）
  - 選択肢のどれも該当しない場合、ユーザーは自動的に表示される **"Other"** を選んで自由テキストを入力できる
  - `options` に "Other" を手動追加する必要はない（ツールが自動付与する）
  - ユーザーが "Other" を選択した場合は、入力内容を要件に反映してから次フェーズへ進む
- **Write / Edit**: `plans/tasks/{タスク名}/prd.md` の作成・更新

## 出力形式

チャットサマリーのみ（ファイルは直接更新）。

```text
## Feature Name: {機能名}
## Purpose: {目的}
## Goal: {ゴール}
## Requirements Review: 不足・矛盾・確定事項
## Updated File: plans/tasks/{タスク名}/prd.md
## Next Step: /steering
```
