# /add-screenshot — spec ファイルの `## 画面` セクションにスクリーンショットを埋め込む

<instructions>

## 役割

spec ファイルの `## 画面` セクションを、実際のスクリーンショット（PNG）で埋め込む。
以下の場合にスクリーンショットを自動撮影する:

- PNG が存在しない
- `sample-api/` または `sample-front/src/` に git diff がある（実装が更新されている）

---

## 実行フロー

### Step 1: 対象 spec ファイルを特定する

- 引数で spec ファイルのパスが渡された場合はそれを使用する
- 渡されなかった場合は `specs/` 配下の `*.md` を列挙し、`AskUserQuestion(multiSelect: true)` で対象を選ばせる

### Step 2: スクリーン名を決定する

- 引数でスクリーン名が渡された場合はそれを使用する（例: `hello`）
- 渡されなかった場合は spec ファイルのパスから推測する（例: `specs/hello/get.md` → `hello`）

### Step 3: `## スクリーンショット設定` が存在するか確認する

spec ファイルを読み込み、`## スクリーンショット設定` セクション（JSON ブロック）が存在するか確認する。

**存在しない場合**: 以下のテンプレートを spec ファイルの末尾に追加して続行する。

```markdown
## スクリーンショット設定

```json
{
  "steps": [
    { "goto": "http://localhost:3000" },
    { "screenshot": "{スクリーン名}" }
  ]
}
```
```

**存在する場合**: そのまま Step 4 へ進む。

### Step 4: 撮影が必要か判定する

以下のいずれかに該当する場合、Step 5 のスクリーンショット撮影を実行する:

| 条件 | 理由 |
|------|------|
| `specs/img/{スクリーン名}.png` が存在しない | 初回撮影が必要 |
| `git diff sample-api/` に差分がある | API の変更で画面が変わっている可能性がある |
| `git diff sample-front/src/` に差分がある | フロントの変更で画面が変わっている可能性がある |
| `git diff specs/img/{スクリーン名}.png` に差分がある | PNG が更新済みだが spec に未反映 |

いずれも該当しない場合は Step 5 をスキップして Step 6 へ進む。

### Step 5: スクリーンショットを撮影する

`sample-front/` ディレクトリで以下を実行する:

```bash
cd sample-front && bun run screenshot
```

- スクリプト: `sample-front/scripts/screenshot.ts`
- 出力先: `specs/img/{スクリーン名}.png`
- サーバー起動: スクリプトが `sample-api`（:8080）と `sample-front`（:3000）を自動起動する
  - 既に起動済みの場合は再利用する
  - 環境変数 `FRONT_PORT` / `API_PORT` で変更可能

**撮影後の確認**:

```bash
git diff --stat specs/img/
```

- PNG が更新されていれば Step 6 へ進む
- 変化なし（スクリプト失敗など）の場合は `AskUserQuestion` で確認する

### Step 6: `## 画面` セクションを更新する

**PNG が存在する場合**（通常ケース）:

```markdown
## 画面

<img src="../../specs/img/{スクリーン名}.png" width="400" alt="{画面名}" />
```

**PNG が存在しない場合**（撮影失敗 / ツールなし）:

spec ファイルの内容（概要・操作フロー）から画面の構成要素を読み取り、ASCII アートを生成して代替表示する。

```markdown
## 画面

<!-- ⚠️ スクリーンショットの生成に失敗しました。PNG が用意できたら /add-screenshot を再実行してください。-->
<!--
┌────────────────────────────────────────────────────────────────┐
│  {画面タイトル}                                                 │
│ ─────────────────────────────────────────────────────────────  │
│  {セクションラベル}                                             │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ {行 1 の内容}                                            │  │
│  │ {行 2 の内容}                                            │  │
│  └──────────────────────────────────────────────────────────┘  │
│  [ {ボタン A} ]  [ {ボタン B} ]                                 │
└────────────────────────────────────────────────────────────────┘
-->
```

### Step 7: `## 画面` セクションが存在しない場合

`## 概要` セクションの直後に `## 画面` セクションを挿入する。

### Step 8: 完了を報告する

---

## スクリーンショット設定の書き方

spec ファイル（例: `specs/hello/get.md`）に以下のセクションを追加する。

````markdown
## スクリーンショット設定

```json
{
  "steps": [
    { "goto": "http://localhost:3000" },
    { "waitForText": "hello" },
    { "screenshot": "hello" }
  ]
}
```
````

### 使えるステップ

| ステップ | 用途 | 例 |
|---------|------|----|
| `goto` | ページに遷移する | `{ "goto": "http://localhost:3000" }` |
| `waitForText` | テキストが表示されるまで待つ | `{ "waitForText": "hello" }` |
| `screenshot` | 現在の画面を撮影する | `{ "screenshot": "hello" }` |
| `click` | 要素をクリックする | `{ "click": { "role": "button", "name": "送信" } }` |
| `fill` | テキストを入力する | `{ "fill": { "role": "textbox", "name": "名前", "value": "太郎" } }` |

---

## 更新ルール

- `## 画面` セクション内のみを置換する。他のセクションは一切変更しない
- 既存の手書き ASCII アートはユーザーが記述したものとみなし、上書きは `AskUserQuestion` で確認を取る
- PNG が生成されたら ASCII アートは削除して `<img>` タグに差し替える

---

## ツール案内

- `Glob` / `Bash`: PNG の存在確認・git diff 実行・`bun run screenshot` 実行
- `Read`: spec ファイルの内容読み取り
- `Edit`: `## 画面` セクションの更新・挿入、`## スクリーンショット設定` の追加
- `AskUserQuestion`: 対象ファイル・スクリーン名の確認、撮影失敗時の対応確認

</instructions>

## 出力形式

```
✅ 画面セクション更新完了
- 対象ファイル: specs/{機能名}/{操作}.md
- スクリーンショット: [撮影済み / スキップ（差分なし） / 失敗（ASCII アートで代替）]
- PNG パス: specs/img/{スクリーン名}.png
- 差分検出: [sample-api / sample-front / なし]
```
