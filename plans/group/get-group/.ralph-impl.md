PRD summary:

Endpoint 1: GET /api/v1/groups/:id
- Path param: id (positive integer)
- Validates: id parseable as int, id >= 1; returns 400 if invalid
- Fetches group from groups table WHERE id=:id AND deleted_at IS NULL; returns 404 if not found
- Response 200: { id, name, description, member_count }
- Errors: 400 (validation), 404 (not found), 500 (db error)
- Error format: { "message": "..." }

Endpoint 2: GET /api/v1/groups/:id/members
- Path param: id (positive integer)
- Query params: limit (optional, 1-500, default 500), offset (optional, 0+, default 0), q (optional search keyword)
- Validates: id parseable/>=1; limit 1-500 if present; offset >=0 if present; returns 400 if invalid
- Checks group exists; returns 404 if not found
- Fetches: group_members JOIN users WHERE group_id=:id, q -> first_name LIKE '%q%' OR last_name LIKE '%q%', LIMIT :limit OFFSET :offset
- total: COUNT(*) for group_id=:id without limit/offset
- Response 200: { members: [{id, first_name, last_name}], total }
- Errors: 400, 404, 500
- Error format: { "message": "..." }

DB Migration (003_create_users_and_update_group_members.sql):
- CREATE TABLE users (id BIGINT UNSIGNED AUTO_INCREMENT PK, first_name VARCHAR(255) NOT NULL, last_name VARCHAR(255) NOT NULL)
- ALTER TABLE group_members: ADD COLUMN user_id BIGINT UNSIGNED NOT NULL REFERENCES users(id), DROP COLUMN name
- Add seed data for users and updated group_members
