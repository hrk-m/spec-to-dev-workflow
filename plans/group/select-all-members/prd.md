# PRD: select-all-members

## 概要

| 項目     | 内容                                                                                         |
| -------- | -------------------------------------------------------------------------------------------- |
| 機能名   | `select-all-members`                                                                         |
| 目的     | MemberList ヘッダーにチェックボックスを追加し、全メンバーを一括選択 / 全解除できるようにする |
| 変更範囲 | フロントエンドのみ（バックエンド API 変更なし）                                              |
| 関連機能 | `remove-group-members`（削除フローはそのまま利用）                                           |

---

## 確認ステップ 5-1: UI インタラクション仕様

### ヘッダーチェックボックスのクリック動作

| 現在の選択状態            | クリック後 |
| ------------------------- | ---------- |
| 全未選択                  | 全選択     |
| indeterminate（一部選択） | 全選択     |
| 全選択                    | 全解除     |

### ヘッダーチェックボックスの表示状態派生

| 条件                             | ヘッダー checkbox の状態 |
| -------------------------------- | ------------------------ |
| メンバー 0 件                    | disabled                 |
| メンバー 1 件以上 かつ 未選択    | 未チェック               |
| 0 < 選択数 < 全体数              | indeterminate            |
| 選択数 = 全体数（かつ 1 件以上） | チェック済み             |
| 削除成功後                       | 全解除（リセット）       |

---

## 確認ステップ 5-2-FE: フロントエンド処理フロー

凡例: `→` = 次の処理へ進む / `終了` = 処理終了

```
1. MemberList がマウントされる
2. メンバー一覧と選択済み ID 一覧から以下の状態を派生する
   - 全選択状態（全員が選択済みかどうか）
   - 一部選択状態（一部のみ選択中かどうか）
   - ヘッダー checkbox の操作可否（メンバーが 0 件なら無効）
3. 副作用フックでヘッダー checkbox の indeterminate 状態を DOM に反映する
   - 全選択状態・一部選択状態が変化するたびに再実行する
4. ユーザーがヘッダー checkbox をクリックする
   - 全選択状態の場合 → 選択状態を全解除する → 終了
   - 全選択状態でない場合（全未選択 or 一部選択）→ 全メンバーを選択状態にする → 終了
5. 選択状態が変わるたびに副作用フックが再実行され indeterminate が再設定される
6. 削除成功後は既存の選択状態リセット処理が走り、
   副作用フック経由でヘッダー checkbox も自動リセットされる → 終了
```

## 要件

1. MemberList テーブルヘッダー左端に「全選択」チェックボックスを追加する
2. 全未選択時にクリックすると表示中の全行を選択する
3. indeterminate（一部選択）時にクリックすると全行を選択する
4. 全選択時にクリックすると全行を選択解除する
5. ヘッダー checkbox の DOM を useRef で所持し、useEffect で indeterminate 属性を直接設定する
6. メンバー 0 件のときヘッダー checkbox を disabled にする
7. 削除成功後はヘッダー checkbox を全未選択状態にリセットする（既存の選択状態リセット処理に連動）
8. 変更範囲はフロントエンドのみ（MemberList.tsx + MemberList.styles.ts）

---

## 対象外

- バックエンド API の変更
- AddMemberSheet の全選択（別機能）
- 無限スクロール中の「現在表示中のみ選択」と「全件選択」の区別（members 配列全体を対象とする）
- ソート順の変更
